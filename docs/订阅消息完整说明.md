# 订阅消息完整说明

## 概述

本文档说明乐学课表小程序的订阅消息功能，包括技术实现、部署配置、重要限制和解决方案。

**核心功能**：在用户设置的上课前 N 分钟，通过微信订阅消息发送上课提醒。

## ⚠️ 重要限制说明

### 微信订阅消息是一次性的！

**这是最重要的一点！**

微信小程序的订阅消息是**一次性授权**，用户授权一次只能发送**一条**消息。发送后授权立即失效，如果要再次发送，需要用户重新授权。

#### 这意味着什么？

1. 用户今天授权 → 今天能收到**一次**提醒
2. 明天想收到提醒 → 需要**重新授权**
3. 每天都要重新授权 → 用户体验较差

#### 当前实现方案

老王我已经在代码里做了以下处理：

1. **发送后标记授权失效**：`cloudfunctions/reminder/index.js` 发送消息后，会将用户的授权状态改为 `'used'`
2. **前端检查授权状态**：通知设置页面会检查授权状态，如果失效会显示警告提示
3. **保存时重新授权**：用户每次保存设置时，会重新请求订阅授权

#### 用户使用流程

```
第1天：
1. 用户开启提醒 → 授权 → 收到提醒 ✅
2. 授权失效 ❌

第2天：
1. 用户打开小程序 → 看到"授权已失效"提示
2. 用户重新进入通知设置 → 点击保存 → 重新授权 → 收到提醒 ✅
3. 授权失效 ❌

第3天：
（重复第2天的流程）
```

## 技术实现

### 模板信息

- **模板ID**: `I1lkvTBguxU146JHrpdlNn9vZA60GBuMPlpW2dSsnK8`
- **用途**: 上课提醒（中午放学 / 下午放学）
- **触发时机**: 用户设置的上课前 N 分钟

### 字段映射

根据微信公众平台模板详情，字段映射如下：

```javascript
{
  thing3: { value: reminder.course_name },                    // 日程描述：课程名称
  time2: { value: reminder.course_time },                     // 日程时间：上课时间（YYYY-MM-DD HH:MM 格式）
  thing6: { value: `${reminder.dismiss_type}提前${reminder.advance_minutes}分钟` }, // 备注：提醒信息
}
```

**注意**：
- `time2` 字段必须是完整的日期时间格式 `YYYY-MM-DD HH:MM`，例如 `2026-04-30 14:30`
- `thing3` 和 `thing6` 字段最多 20 个字符

### 提醒逻辑

#### 中午放学提醒

- 找出该课表当天**上午最后一节课**（slot <= 5 中最大的）
- 在该课程开始前 N 分钟发送提醒
- 用户可以在设置中关闭中午放学提醒

#### 下午放学提醒

- 找出该课表当天**下午最后一节课**（slot >= 6 中最大的）
- 在该课程开始前 N 分钟发送提醒
- 用户可以在设置中关闭下午放学提醒

#### 提前时间

- 用户可以设置提前 5/10/15/20/25 分钟
- 默认提前 10 分钟

#### 默认时间表

`cloudfunctions/reminder/generate.js` 中的 `getSlotTime()` 函数定义了每节课的开始时间：

- 第1节：08:00
- 第2节：08:50
- 第3节：09:50
- 第4节：10:40
- 第5节：11:30
- 第6节：14:00
- 第7节：14:50
- 第8节：15:40
- 第9节：16:30
- 第10节：17:20

### 数据库设计

#### reminders 集合

提醒记录集合，每天凌晨 00:05 生成当天的提醒记录。

**字段说明**：
- `openid`: 接收者 openid
- `schedule_id`: 课表ID
- `course_id`: 课程ID
- `course_name`: 课程名称
- `course_time`: 上课时间（YYYY-MM-DD HH:MM 格式）
- `room`: 上课地点
- `slot`: 节次
- `advance_minutes`: 提前分钟数
- `trigger_time`: 触发时间
- `status`: 状态（pending / sent / skipped / failed）
- `retry_count`: 重试次数
- `date`: 日期（YYYY-MM-DD）
- `dismiss_type`: 放学类型（中午放学 / 下午放学）
- `sent_at`: 发送时间
- `skip_reason`: 跳过原因（no_subscribe / user_disabled）
- `last_error`: 最后错误信息

#### users.subscribe_tokens

用户��阅授权记录，存储在 `users` 集合的 `subscribe_tokens` 数组字段中。

**字段说明**：
- `template_id`: 模板ID
- `result`: 授权结果（accept / reject / ban / used）
- `updated_at`: 更新时间
- `used_at`: 使用时间（仅 result='used' 时有值）

### 代码修改说明

#### 1. 云函数标记授权失效

`cloudfunctions/reminder/index.js` 第 148-159 行：

```javascript
if (sent) {
  // 发送成功后，标记授权已使用
  const userToUpdate = await db.findOne('users', { openid: reminder.openid });
  if (userToUpdate && userToUpdate.subscribe_tokens) {
    const updatedTokens = userToUpdate.subscribe_tokens.map(t => {
      if (t.template_id === TEMPLATE_ID && t.result === 'accept') {
        return { ...t, result: 'used', used_at: new Date() };
      }
      return t;
    });
    await db.update('users', userToUpdate._id, { subscribe_tokens: updatedTokens });
  }
  successCount++;
}
```

#### 2. 新增授权状态检查API

`cloudfunctions/notify/index.js` 新增 `checkSubscribeStatus` action：

```javascript
async function checkSubscribeStatus(openid, payload) {
  const user = await db.findOne('users', { openid });
  const subscribeTokens = user.subscribe_tokens || [];
  const token = subscribeTokens.find(t => t.template_id === payload.templateId);
  const hasValidAuth = token && token.result === 'accept';
  
  return success({
    hasValidAuth,
    lastResult: token ? token.result : null,
    lastUpdateTime: token ? token.update_time : null,
  });
}
```

`taro-app/src/pages/notification-settings/index.tsx`：

- 如果授权失效，显示橙色警告提示
- 用户保存设置时重新请求授权

```bash
# 部署 reminder 云函数（提醒生成和发送）
npm run deploy:reminder

# 部署 notify 云函数（订阅授权记录和状态检查）
npm run deploy:notify

# 部署 auth 云函数（订阅授权记录）
npm run deploy:auth
```

### 2. 配置定时触发器

在微信云开发控制台配置定时触发器：

**触发器1：生成提醒记录**
- 触发周期：`0 5 0 * * * *`（每天 00:05）
- 触发参数：`{ "action": "generate" }`

**触发器2：发送提醒消息**
- 触发周期：`0 * * * * * *`（每分钟）
- 触发参数：`{ "action": "send" }` 或 `{}`

### 3. 前端构建和上传

```bash
cd taro-app
pnpm build:weapp
pnpm ci:upload
```

2. 开启某个学生的"中午放学"或"下午放学"提醒

4. 点击"允许"授权
5. 等待提醒时间到达，检查是否收到订阅消息

## 解决方案

### 方案1：申请长期订阅（强烈推荐）

微信小程序有**长期订阅**功能，用户授权一次后可以长期发送（有频率限制）。

**适用类目**：
- 教育类（在线教育、教育信息服务）
- 医疗类
- 政务类
- 交通类
- 金融类

**你的课表小程序属于教育类，可以申请！**

**申请步骤**：

1. 登录 [微信公众平台](https://mp.weixin.qq.com)
2. 进入"设置 → 基本设置 → 服务类目"
3. 添加类目：
   - 一级类目：教育
   - 二级类目：在线教育 或 教育信息服务
4. 提交资质审核（可能需要）：
   - 教育类资质证明
   - 营业执照
   - ICP备案
5. 审核通过后，进入"功能 → 订阅消息"
6. 选择"长期订阅"模板（而不是"一次性订阅"）
7. 申请模板，获取新的模板ID
8. 替换代码中的模板ID

**长期订阅的限制**：
- 每个用户每天最多发送 1-3 次（根据类目不同）
- 用户可以随时取消订阅
- 需要定期检查用户是否取消订阅

### 方案2：每周提醒（折中方案）

既然微信订阅消息是一次性的，那就改成**每周提醒一次**，而不是每天提醒。

**修改思路**：

在通知设置页面，让用户选择提醒频率：
- 每天提醒（需要每天授权）
- 每周一次（每周授权一次）
- 每周两次（每周授权两次）

**修改生成逻辑**（`cloudfunctions/reminder/generate.js`）：

```javascript
// 检查用户的提醒频率设置
const reminderFrequency = settings.reminder_frequency || 'daily'; // daily / weekly / twice_weekly

// 如果是每周提醒，只在周一生成
if (reminderFrequency === 'weekly') {
  const today = new Date();
  const dayOfWeek = today.getDay(); // 0=周日，1=周一
  if (dayOfWeek !== 1) {
    continue; // 不是周一，跳过
  }
}

// 如果是每周两次，只在周一和周四生成
if (reminderFrequency === 'twice_weekly') {
  const today = new Date();
  const dayOfWeek = today.getDay();
  if (dayOfWeek !== 1 && dayOfWeek !== 4) {
    continue; // 不是周一或周四，跳过
  }
}
```

**优点**：
- 授权频率低，用户不烦
- 仍然能起到提醒作用
- 符合微信订阅消息的限制

**缺点**：
- 只能每周提醒一次，不是每天
- 如果用户忘记授权，整周都收不到提醒

### 方案3：保持现状 + 优化体验

如果无法申请长期订阅，可以优化用户体验：

1. **在首页显示授权状态**：提醒用户授权已失效
2. **简化授权流程**：一键重新授权
3. **定期提醒用户授权**：每周提醒一次
4. **说明文档**：在小程序内说明订阅消息的限制

**建议在小程序内明确告知用户**：

> "由于微信订阅消息限制，每次授权只能发送一次提醒。如需持续接收提醒，请定期重新授权。"

### 方案4：改用其他通知方式

如果订阅消息无法满足需求，可以考虑：

1. **短信通知**（需要接入短信服务商，有成本）
2. **邮件通知**（需要用户提供邮箱）
3. **企业微信通知**（如果用户是企业用户）
4. **小程序内消息**（用户需要打开小程序才能看到）

## 常见问题

### 1. 用户没有收到消息

**检查清单**：
- 用户是否同意了订阅授权（`users.subscribe_tokens` 中 `result` 是否为 `'accept'`）
- 用户是否开启了通知（`users.settings.notify_enabled` 是否为 `true`）
- 用户是否开启了对应时段的提醒（`student_settings[studentId].noon_enabled` / `afternoon_enabled`）
- 提醒记录状态（`reminders` 集合中 `status` 字段）
- 定时触发器是否正常运行

### 2. 发送失败

**排查步骤**：
- 查看云函数日志，确认错误信息
- 检查模板ID是否正确
- 检查字段名是否与模板一致（thing3 / time2 / thing6）
- 检查字段值是否超过长度限制（thing 类型最多 20 字符）
- 检查 `course_time` 格式是否正确（应为 `YYYY-MM-DD HH:MM` 格式）

### 3. 订阅授权失败

**可能原因**：
- 小程序未发布（体验版和开发版无法使用订阅消息）
- 模板ID未在微信公众平台添加
- 小程序未关联微信公众平台账号
- 用户点击了"拒绝"或"总是拒绝"

### 4. 提醒时间不准确

**检查**：
- `cloudfunctions/reminder/generate.js` 中的 `getSlotTime()` 函数
- 该函数定义了每节课的开始时间，需要根据学校实际情况调整

## 监控和维护

### 查看提醒记录

```javascript
// 查询今天的提醒记录
db.collection('reminders')
  .where({
    date: '2026-04-30'
  })
  .get()

// 查询发送失败的提醒
db.collection('reminders')
  .where({
    status: 'failed'
  })
  .get()
```

### 查看用户订阅状态

```javascript
// 查询已授权的用户
db.collection('users')
  .where({
    'subscribe_tokens.result': 'accept'
  })
  .get()
```

### 清理过期提醒记录

建议定期清理 7 天前的提醒记录，避免数据库膨胀：

```javascript
// 删除 7 天前的提醒记录
const sevenDaysAgo = new Date()
sevenDaysAgo.setDate(sevenDaysAgo.getDate() - 7)

db.collection('reminders')
  .where({
    trigger_time: db.command.lt(sevenDaysAgo)
  })
  .remove()
```

## 建议

**强烈建议申请长期订阅！**

课表提醒是典型的教育类应用场景，完全符合长期订阅的要求。申请通过后，用户体验会大幅提升，不需要每天重新授权。

如果暂时无法申请，可以考虑"每周提醒"方案作为折中，或者保持现状并优化用户体验。
